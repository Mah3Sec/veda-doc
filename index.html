Introduction
VAJRA — Vulnerability Analysis, Judgement & Reporting Arsenal

VAJRA is a locally-hosted AI report generator for security professionals. It transforms your raw pentest findings into complete, professional reports — aligned with CREST, PTES, OWASP, and NIST standards — using any AI provider you configure.

Everything runs on your machine. Your findings and client data never leave your network except to the AI endpoint you explicitly configure.

Features

3 Report TypesPenetration Test, Red Team Engagement, Phishing Campaign
Any AI ProviderClaude, ChatGPT, Gemini, Grok, Groq, Ollama and more
100% LocalRuns on localhost — data stays on your machine
Team ReadyMulti-user with admin/analyst/viewer roles and report sharing
Tool ImportNessus .nessus, Burp Suite .xml, Nmap .xml auto-parsed
DOCX + PDF ExportProfessional styled Word docs with logo embedding
Custom TemplatesUpload your own branded .docx or .md templates
Large AssessmentsParallel chunked generation for 10+ findings without timeouts

Report Standards

Every generated report is structured to meet professional security assessment standards:

  • CREST — penetration testing and red team methodology
  • PTES — Penetration Testing Execution Standard
  • OWASP WSTG v4.2 — Web Security Testing Guide
  • NIST SP 800-115 — Technical Guide to Security Testing
  • CVSS v3.1 — Common Vulnerability Scoring System
  • MITRE ATT&CK — adversarial tactics and techniques (Red Team)

Quick Install (recommended)

The setup script handles everything — dependencies, config, and directories.

git clone https://github.com/Mah3Sec/VAJRA.git
cd VAJRA
bash setup.sh

What the setup script does:

  • Verifies Python and Node.js are installed
  • Runs pip3 install -r requirements.txt
  • Runs npm install in docx_builder/
  • Copies .env.example → .env
  • Creates reports/, database/, ui/static/logos/ directories

Manual Install

# Python dependencies
pip3 install -r requirements.txt --break-system-packages

# Node.js dependencies (for DOCX builder)
cd docx_builder && npm install && cd ..

# Config file
cp .env.example .env
nano .env   # add your AI_API_KEY and AI_PROVIDER

# Runtime directories
mkdir -p reports database ui/static/logos

First Run

python3 app.py

Open http://localhost:5000 in your browser.

Default credentials: username admin / password vajra-admin-2026
Change the admin password immediately after first login — Settings → Profile.

Docker

docker build -t vajra .
docker run -p 5000:5000 --env-file .env vajra
Quick Start
Generate your first professional report in 5 steps.
1

Configure your AI provider

Edit .env and set your provider, API key, and model. Example for Anthropic:

AI_PROVIDER=anthropic
AI_API_KEY=sk-ant-your-key-here
AI_BASE_URL=https://api.anthropic.com
AI_MODEL=claude-sonnet-4-6

See AI Providers for configuration examples for every supported provider.

2

Start VAJRA

python3 app.py

Open http://localhost:5000 and log in with your admin credentials.

3

Select report type and fill engagement details

Choose between Penetration Test, Red Team, or Phishing Campaign. Fill in client name, lead assessor, scope, date, and any type-specific fields.

4

Add your findings

Three methods available — paste raw notes, fill the structured form, or upload a scanner export file (Nessus, Burp Suite, Nmap XML).

5

Generate and export

Click Generate Report. The AI builds the full report. Export as DOCX, PDF, or Markdown when done.

Large assessments: For 10+ findings, VAJRA automatically switches to parallel chunked generation. A real-time progress bar tracks each batch — no gateway timeouts.

Report Types
VAJRA generates three types of professional security assessment reports.

Penetration Test Report

The standard pentest report type. Follows CREST / PTES / OWASP WSTG v4.2 / NIST SP 800-115 methodology. Uses CVSS v3.1 scoring with calculated remediation deadlines (Critical+2d, High+7d, Medium+30d, Low+90d).

SectionContents
Executive SummaryBoard-level overview, overall risk rating, findings count by severity, most critical action
Engagement OverviewScope, methodology phases, CVSS risk rating table, assessment limitations
Findings SummaryFull vulnerability table with CVSS scores, severity distribution breakdown
Detailed FindingsPer finding: CVSS v3.1, CWE, OWASP Top 10 mapping, steps to reproduce, evidence, business impact, remediation steps
Remediation RoadmapPrioritised schedule with calculated calendar deadlines per finding
AppendixTools used, regulatory mapping (PCI DSS, GDPR, ISO 27001), legal disclaimer, references

Red Team Engagement Report

For full adversary simulation engagements. Follows CREST STAR / CBEST / TIBER-EU standards. Includes MITRE ATT&CK TTP mapping and blue team detection assessment.

SectionContents
Executive SummaryObjective outcome, whether crown jewels were reached, detection rate, dwell time
Engagement OverviewObjective, threat actor profile simulated, scope, methodology phases
Attack NarrativeFull chronological kill chain with timestamps, techniques, and pivots
TTP ObservationsPer-TTP blocks with MITRE ATT&CK mapping, evidence, detection gaps, recommendations
MITRE ATT&CK MappingFull table of tactics, techniques, and tools used across the engagement
Blue Team AssessmentDetection timeline per phase, what was missed, SIEM rule and log source recommendations

Phishing Campaign Report

For simulated phishing awareness exercises. Constructive, educational tone — not an offensive attack report. Follows NIST / SANS / CREST standards.

SectionContents
Executive SummaryOverall susceptibility rate, key behavioural finding, recommended action
Campaign MetricsEmails sent, delivered, opened, clicked, credentials submitted, MFA bypassed, reported by staff
Department BreakdownPer-department click and submission rates with risk level
Email Security ControlsSPF, DKIM, DMARC configuration and effectiveness assessment
Phishing Resilience ScoreCalculated score (non-click rate × 0.4 + report rate × 0.6) with benchmark comparison
RecommendationsTargeted training by risk group, technical controls, awareness programme cadence
Adding Findings
Three methods to get your findings into VAJRA. Mix and match as needed.

Method 1 — Paste Raw Notes

The fastest method. Dump your raw assessment notes directly into the findings box — no formatting required. VAJRA's AI structures everything into proper finding blocks with CVSS, CWE, remediation, and all required sections.

SQL injection on /api/login — high severity
The username parameter is vulnerable to blind boolean-based SQLi.
Confirmed with sqlmap: extracted database credentials.
Database: MySQL 8.0, running as root.

Reflected XSS in search — medium
Parameter: q
Payload: <script>alert(document.cookie)</script>
Tested on Chrome 121, no CSP headers present.

Missing security headers — low
No X-Frame-Options, X-Content-Type-Options, or Content-Security-Policy headers.

Method 2 — Manual Structured Form

Fill each finding individually using structured input fields: title, severity (Critical/High/Medium/Low/Info), description, steps to reproduce, business impact, remediation, CVSS score, and references. Best for precise control over each finding's content and for ensuring complete data before generation.

Method 3 — Tool Import

Upload an export file directly from your scanning tool. VAJRA auto-detects the format, parses all findings, and converts them to structured text ready for AI generation.

ToolFile FormatHow to Export
Nessus.nessusScans → select scan → Export → Nessus format
Burp Suite Pro.xmlIssue activity → Report → XML → Export
Nmap.xmlRun with nmap -oX output.xml [target]
Large assessments (10+ findings): VAJRA automatically detects the finding count and switches to parallel chunked generation. A progress bar tracks each batch completing in real time. No gateway timeouts.

Install PDF Export

VAJRA tries all available PDF backends automatically in order: LibreOffice → pandoc+wkhtmltopdf → pandoc+LaTeX → weasyprint. Install any one and it will be used.

Recommended — easiest to install, works well:

pip3 install weasyprint markdown --break-system-packages

Best quality output — converts the fully styled DOCX:

brew install --cask libreoffice    # macOS
sudo apt install libreoffice       # Debian / Ubuntu

Team Deployment
Run VAJRA on a shared internal server so your whole team can access it.

Docker (recommended)

# Build the image
docker build -t vajra .

# Run with persistent data volumes
docker run -d \
  --name vajra \
  --restart unless-stopped \
  -p 5000:5000 \
  --env-file .env \
  -v $(pwd)/reports:/app/reports \
  -v $(pwd)/database:/app/database \
  -v $(pwd)/ui/static/logos:/app/ui/static/logos \
  vajra

nginx Reverse Proxy with HTTPS

Put VAJRA behind nginx for HTTPS. The proxy_read_timeout is important — report generation can take 30–120 seconds for large assessments.

server {
    listen 443 ssl http2;
    server_name vajra.yourteam.com;

    ssl_certificate     /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_set_header   Host $host;
        proxy_set_header   X-Real-IP $remote_addr;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }
}

Set these in .env when behind HTTPS:

HTTPS=true
HOST=127.0.0.1
PORT=5000
SECRET_KEY=your-strong-random-key-here

Security Implementation

  • Password hashing: PBKDF2-HMAC-SHA256 with 260,000 iterations and a per-user random salt
  • Brute-force protection: 10 failed attempts per IP address or username triggers a 5-minute lockout
  • Session fixation: Session cleared before new login — prevents session fixation attacks
  • Secure cookies: HttpOnly, SameSite=Strict, optional Secure flag when HTTPS=true
  • Rate limiting: Maximum 10 report generations per 5 minutes per user
  • File upload limits: 50MB maximum upload size
Adding a Provider
Extend VAJRA with a new AI provider. Most take under 5 minutes to add.

Option A — OpenAI-compatible endpoint (1 line)

Most modern AI APIs implement the OpenAI /v1/chat/completions wire format. If your provider does, just add its base URL to _OPENAI_COMPAT_URLS in app.py:

# In app.py — find _OPENAI_COMPAT_URLS and add your provider:
_OPENAI_COMPAT_URLS = {
    "openai":      "https://api.openai.com/v1",
    "grok":        "https://api.x.ai/v1",
    # ... existing providers ...
    "myprovider":  "https://api.myprovider.com/v1",   # ← add this line
}

Then set in .env:

AI_PROVIDER=myprovider
AI_API_KEY=your-key
AI_MODEL=their-model-name

Done — no other code changes needed. VAJRA will route all API calls through your provider automatically.

Option B — Custom API format

If your provider uses a unique request format (like Google Gemini), add an elif branch in the _call_provider() function in app.py:

elif p == "myprovider":
    # Import the provider's SDK
    import myprovider_sdk
    client = myprovider_sdk.Client(api_key=api_key)
    response = client.generate(
        model=model,
        system=system_prompt,
        prompt=user_prompt,
        max_tokens=max_tokens,
    )
    return response.text

Document your provider

After adding a new provider, update these files so others can use it:

  • .env.example — add a commented configuration block
  • README.md — add a row to the providers table
Contribute: Open a pull request on GitHub to get your provider included in the next release.

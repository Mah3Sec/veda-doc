Introduction
VAJRA — Vulnerability Analysis, Judgement & Reporting Arsenal

VAJRA is a locally-hosted AI report generator for security professionals. It transforms your raw pentest findings into complete, professional reports — aligned with CREST, PTES, OWASP, and NIST standards — using any AI provider you configure.

Everything runs on your machine. Your findings and client data never leave your network except to the AI endpoint you explicitly configure.

Features

3 Report TypesPenetration Test, Red Team Engagement, Phishing Campaign
Any AI ProviderClaude, ChatGPT, Gemini, Grok, Groq, Ollama and more
100% LocalRuns on localhost — data stays on your machine
Team ReadyMulti-user with admin/analyst/viewer roles and report sharing
Tool ImportNessus .nessus, Burp Suite .xml, Nmap .xml auto-parsed
DOCX + PDF ExportProfessional styled Word docs with logo embedding
Custom TemplatesUpload your own branded .docx or .md templates
Large AssessmentsParallel chunked generation for 10+ findings without timeouts

Report Standards

Every generated report is structured to meet professional security assessment standards:

  • CREST — penetration testing and red team methodology
  • PTES — Penetration Testing Execution Standard
  • OWASP WSTG v4.2 — Web Security Testing Guide
  • NIST SP 800-115 — Technical Guide to Security Testing
  • CVSS v3.1 — Common Vulnerability Scoring System
  • MITRE ATT&CK — adversarial tactics and techniques (Red Team)

Quick Install (recommended)

The setup script handles everything — dependencies, config, and directories.

git clone https://github.com/Mah3Sec/VAJRA.git
cd VAJRA
bash setup.sh

What the setup script does:

  • Verifies Python and Node.js are installed
  • Runs pip3 install -r requirements.txt
  • Runs npm install in docx_builder/
  • Copies .env.example → .env
  • Creates reports/, database/, ui/static/logos/ directories

Manual Install

pip3 install -r requirements.txt --break-system-packages
cd docx_builder && npm install && cd ..
cp .env.example .env
nano .env
mkdir -p reports database ui/static/logos

First Run

python3 app.py

Open http://localhost:5000 in your browser.

Default credentials: username admin / password vajra-admin-2026
Change the admin password immediately after first login.

Docker

docker build -t vajra .
docker run -p 5000:5000 --env-file .env vajra
Quick Start
Generate your first professional report in 5 steps.
1

Configure your AI provider

Edit .env and set your provider, API key, and model:

AI_PROVIDER=anthropic
AI_API_KEY=sk-ant-your-key-here
AI_BASE_URL=https://api.anthropic.com
AI_MODEL=claude-sonnet-4-6
2

Start VAJRA

python3 app.py

Open http://localhost:5000 and log in.

3

Select report type and fill engagement details

Choose Penetration Test, Red Team, or Phishing Campaign. Fill in client name, assessor, scope, and date.

4

Add your findings

Paste raw notes, fill the structured form, or upload a scanner export (Nessus, Burp Suite, Nmap XML).

5

Generate and export

Click Generate Report. Export as DOCX, PDF, or Markdown when done.

Large assessments: For 10+ findings, VAJRA automatically switches to parallel chunked generation with a real-time progress bar.

Adding Findings
Three methods to get findings into VAJRA.

Method 1 — Paste Raw Notes

Dump your raw notes directly. VAJRA structures everything into proper findings with CVSS, CWE, and remediation.

SQL injection on /api/login — high severity
Confirmed with sqlmap. Database: MySQL 8.0, running as root.

Reflected XSS in search — medium
Parameter: q — no CSP headers present.

Missing security headers — low
No X-Frame-Options or Content-Security-Policy.

Method 2 — Manual Structured Form

Fill each finding individually: title, severity, description, steps to reproduce, business impact, remediation, CVSS score, and references.

Method 3 — Tool Import

Team Deployment
Run VAJRA on a shared internal server.

Docker

docker build -t vajra .
docker run -d --name vajra --restart unless-stopped \
  -p 5000:5000 --env-file .env \
  -v $(pwd)/reports:/app/reports \
  -v $(pwd)/database:/app/database \
  vajra

nginx Reverse Proxy

server {
    listen 443 ssl http2;
    server_name vajra.yourteam.com;
    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_read_timeout 300s;
    }
}

Set in .env when behind HTTPS:

HTTPS=true
HOST=127.0.0.1
SECRET_KEY=your-strong-random-key-here

Backup

tar -czf vajra-backup-$(date +%Y%m%d).tar.gz database/ reports/

Security Implementation

  • Password hashing: PBKDF2-HMAC-SHA256, 260,000 iterations, per-user salt
  • Brute-force protection: 10 attempts per IP/username → 5-minute lockout
  • Secure cookies: HttpOnly, SameSite=Strict, optional Secure flag
  • Rate limiting: 10 report generations per 5 minutes per user
Adding a Provider
Extend VAJRA with a new AI provider in under 5 minutes.

Option A — OpenAI-compatible (1 line)

_OPENAI_COMPAT_URLS = {
    # ... existing providers ...
    "myprovider": "https://api.myprovider.com/v1",
}
AI_PROVIDER=myprovider
AI_API_KEY=your-key
AI_MODEL=their-model-name

Option B — Custom API format

elif p == "myprovider":
    import myprovider_sdk
    client = myprovider_sdk.Client(api_key=api_key)
    response = client.generate(model=model, prompt=user_prompt)
    return response.text
